Consumindo o serviço de autorização de acesso a um recurso

Visão Geral​

O serviço de autenticação & autorização efetua todo processo de Login de um sistema, retornando todas as permissões do usuário em um Sistema ao final do processo.

O Sistema cliente pode desejar um controle mais fino dos acessos, registrando a data e hora de acesso de um usuário a cada Recurso do Sistema em uma sessão de trabalho.

Para verificar se o Usuário possui acesso a um recurso específico e registrar data e hora de acesso a esse recurso o serviço de autorização de acesso a um recurso deve ser consumido.

Esse serviço retorna um código HTTP 200 se o usuário possuir acesso ao recurso.

Para realizar a requisição a uri de autorização de acesso a um recurso, o provedor e o consumidor devem ser iguais. Somente o próprio sistema como Consumidor possui direito de executar esse serviço na API.

Para consumir o serviço, você deve efetuar uma requisição HTTP GET para a URI https://jeap.rio.rj.gov.br/cerberus/seam/resource/v1/permissoes/CHAVE_DO_RECURSO onde CHAVE_DO_RECURSO deve ser substituído pela chave do Recurso que você quer verificar a permissão. Informe também os seguintes headers na requisição.

NomeDescrição
ambienteAmbiente que deseja efetuar a requisição DESENVOLVIMENTO, HOMOLOGACAO ou PRODUCAO sem acentos ou cedilha.
provedorIdentificador do Sistema que você deseja validar as permissões do Usuário.
consumidorIdentificador do Consumidor(Sistema ou Usuário) que está efetuando a chamada à API do Cerberus.
chaveAcessoChave de Acesso do Consumidor.
usuarioUsuário que terá a permissão verificada no Recurso.
recursoRecurso desejado para verificação.

Exemplo em Java para consumo do Serviço​

Vamos demonstrar como efetuar o consumo do serviço atarvés da plataforma Java.

HttpClient client = HttpClientBuilder.create().build();
        
        HttpGet request = new HttpGet("https://jeap.rio.rj.gov.br/cerberus/seam/resource/v1/permissoes/NOME_RECURSO");
        
        request.addHeader("ambiente", "DESENVOLVIMENTO");
    
        request.addHeader("provedor", "EXEMPLO");
    
        request.addHeader("consumidor", "EXEMPLO");
        
        request.addHeader("chaveAcesso", "d4fec2a2-7f41-43bd-9c0c-c00afe8f4858");
    
        request.addHeader("usuario", "26218088");
    
        HttpResponse response = client.execute(request);
        
        System.out.println(response.getStatusLine().getStatusCode());

Qualquer Plataforma que tenha recursos para enviar uma requisição HTTP pode consumir a API do Cerberus.
