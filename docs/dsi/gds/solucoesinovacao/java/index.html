História​

2007-2008​

O início​

Por conta da criticidade e urgência do Projeto Convênios, que tinha como objetivo desenvolver um sistema corporativo de convênios para a Prefeitura, a Iplanrio precisou reunir diversos profissionais em Java. Desenvolvedores de diversas secretarias foram alocados e desse movimento surgiu a Equipe Java.

No final de 2007, o Projeto Convênio foi cancelado. A equipe reunida se organizou e iniciou o desenvolvimento de projetos para diversas secretarias. Não existia um padrão de desenvolvimento estabelecido.

Nesse momento foi iniciado o movimento ágil dentro da Equipe Java.

No início de 2008, o primeiro projeto desenvolvido pela equipe, seguindo os principios da agilidade, foi colocado em produção.

Projetos Desenvolvidos​

  • MEI – Metas & Indicadores

2009​

Padronização​

Com a necessidade de uma padronização do desenvolvimento Java na Prefeitura, tanto em tecnologias quanto em padrões de desenvolvimento, a equipe Java criou a plataforma Maverick.

Maverick 1.0​

A primeira versão do Maverick 1.0 disponibiliza uma documentação com os padrões, arquiteturas e tecnologias que deveriam ser utilizadas. Existia um gerador que criava o projeto java com os frameworks, padrões e bibliotecas necessários.

Tecnologias​

  • Java 6
  • Jboss Community 4.2.3
  • Java EE 5 (JSF 1.0, EJB 3.0, JPA 1.0, Primefaces 1.1, Rich Faces 3.3)
  • Jboss Seam 2.0
  • Ant
  • SVN
  • Jenkins

O Jboss Seam foi definido como framework padrão para desenvolvimento por ser um framework full stack que integrava todos os componentes da especificação Java EE 5, trazendo diversas vantagens e facilidades para desenvolvimento nessa plataforma.

Projetos desenvolvidos​

  • ANTARES – Agendamento do Planetário – Planetário
  • SBJR – Bancas & Jornal – SEOP
  • Pleitos – Sistema de Votação

Projetos Novos/Manutenção​

A equipe recebeu novos desenvolvedores.

Projetos desenvolvidos​

  • CFC – Cartão Família Carioca
  • SISORT – Sistema de Sorteio
  • Vacinas
  • DARMGENERICO – Darm Generico
  • SAVI – Avaliação de Imóveis
  • SCUFF – Controle de Feiras e Feirantes
  • GESTVO – Gestão de Vilas Olimpicas
  • SISPORT – Sistema de Portarias

2011-2012​

Reuso​

A equipe estava desenvolvendo aplicações para diversas secretarias e estava mais organizada. Existia um padrão estabelecido, mas ainda existam problemas como, duplicação de código entre os projetos e não existiam abstrações para minimizar estas questões. Maverick 2.0 Com a necessidade da criação de mais abstrações para reaproveitamento do código entre os projetos, uma nova versão do Maverick foi disponibilizada. As bibliotecas e versões do Java e Jboss foram atualizadas.

Tecnologia​

  • Java 7
  • Jboss EAP 5.1.2
  • Java EE 5 (JSF 1.0, EJB 3.0, JPA 1.0, Primefaces 1.1, Rich Faces 3.3)
  • Jboss Seam 2.2
  • Ant
  • SVN
  • Jenkins

Projetos Desenvolvidos​

  • BEBECARIOCA – Bebê Carioca
  • DESESC – Desempenho Escolar

2013​

Migração​

Os novos projetos foram criados utilizando o Jboss Seam 2.3 e a plataforma Maverick foi atualizada para sua versão 3.0.

Maverick 3.0​

A plataforma Maverick 3.0 não é completa como as anteriores. Essa versão não possui site com documentação e não possui gerador de projetos. Foi disponibilizado algumas classes de apoio para facilitar o desenvolvimento.

Tecnologias​

  • Java 8
  • Jboss EAP 6.4
  • Java EE 6 (JSF 2.0, EJB 3.1, JPA 2.0, Primefaces 5.x)
  • Jboss Seam 2.3
  • Ant
  • SVN
  • Jenkins

2014-2015​

Autenticação/Autorização​

Todo o fluxo de autenticação e autorização era duplicado entre os projetos e não existia um sistema para autenticação e autorização centralizado. Todo sistema possui sua própria tabela de Usuário, Perfil e Usuario_Perfil. Por conta desta situação, surgiu a necessidade de se criar um sistema centralizado de autenticação/autorização interno. Cerberus O Cerberus foi desenvolvido com o objetivo de controlar a autenticação dos usuários usando um dos mecanismos de autenticação corporativos da Prefeitura:

  • Riomail (OpenLdap)
  • Domínio Rio (Active Directory Microsoft)
  • Novell (Somente para Secretaria de Fazenda)

Para autorização foram criadas tabelas de Perfil, Recursos e Usuários dentro do Cerberus e todos os dados de autorização ficam nessas tabelas. Para os sistemas utilizarem o Cerberus eles precisam efetuar uma chamada ao serviço Rest de Permissões. Esse serviço primeiro autentica o usuário em uma das árvores e após a autenticação ser realizada com sucesso efetua a autorização do funcionário com os dados cadastrados nas tabelas de autorização. Retornando as informações no formato JSON para o consumidor do Serviço.

Reestruturação Processo Ágil​

Diante da quantidade de aplicações sendo desenvolvidas pela equipe e projetos novos surgindo, a equipe sentiu a necessidade de se reorganizar para conseguir manter os sistemas legados e desenvolver novas aplicações. Todo o processo de reestruturação foi realizado de forma colaborativa e deste movimento surgiu o Processo de Desenvolvimento Ágil.

Projetos Desenvolvidos​

  • Cerberus
  • RUC – Registro Único do Cidadão
  • Notificações (Envio de emails e mensagens)

2016​

O fim do Jboss Seam​

O projeto JBoss Seam foi perdendo força ao longo do tempo e em 2016 passou a não receber a atenção devida pela comunidade e não houve mais atualizações e evoluções.

Nesse cenário os projetos começaram a ser construídos sem o framework utilizando JAVAEE 7, como especificação principal.

Projetos desenvolvidos​

  • Contracheque

2017-2018​

Taxi.rio​

A plataforma taxi.rio foi criada na PCRJ, sendo a primeira experiência em cloud da PCRJ. O APP foi escrito em Kotlin e Swift e o backend do app foi desenvolvido por uma equipe externa em node.js. Essa estrutura de backend é executada na Azure da Microsoft. A Equipe Java foi acionada para efetuar a criação dos sistemas de suporte a plataforma, que são: BackOffice, Corporativo e Financeiro. Como já estavam acontecendo pesquisas em novas tecnologias, a criação desses sistemas foi fundamental para consolidar essa nova estrutura. Não houve um atualização do Maverick e os projetos seguiram a especificação Java sem um framework de apoio. Esses projetos não possuem uma documentação da sua Arquitetura e Padrões nem uma ferramenta para criação de novos projetos.

Tecnologias​

  • Maven: Os projetos agora são criados com Maven substituindo ant como ferramenta de build.
  • GIT: Todos os projetos da Equipe Java foram migrados para o GitLab e não usamos mais SVN para nenhum projeto.
  • GITLAB CI: Os projetos novos agora não utilizam mais Jenkins e sim GitLab Pipeline para efetuar o ciclo de deploy.
  • Docker: A partir de 2017, todos os projetos utilizam Docker. Novos projetos possuem um docker-compose que configura todo o ambiente do projeto na máquina do desenvolvedor sem a necessidade de criação e configuração manual.
  • CLuster Docker Swarm: Apenas os projetos Backoffice, Corporativo e Financeiro são executados na nuvem da Azure em um CLuster Docker Swarm para orquestração do container.
  • Azure: A equipe Java efetuou a criação da infraestrutura em nuvem do taxi.rio na Azure. Utilizamos terraform para automatizar a infraestrutura e Docker Swarm como orquestrador de container.

Projetos Desenvolvidos​

  • GDF – Gestão de Desenvolvimento de Fábricas
  • SISLOJA – Obras Online
  • CARIOCALITE – Atendimento do Carioca Digital
  • Taxi Rio Backoffice
  • Taxi Rio Corporativo
  • Taxi Rio Financeiro
  • Ambulante Legal

2019​

.rio​

No início de 2019 foi solicitada a criação de um aplicativo para disponibilizar serviços da Secretaria de Saúde aos cidadãos, o saude.rio. Com esse projeto começou a se pensar em uma ferramenta similar ao Cerberus para ser uma ferramenta de centralização e Single Sign-On das informações de autenticação para os cidadãos.

A ferramenta escolhida foi o Keycloak para servir como centralizador dos dados de autenticação dos cidadãos ao serviços novos criados pela PCRJ. Ao longo do ano de 2019, o uso do Keycloak se solidificou e ele se tornou o Identidade Carioca, plataforma responsável pela centralização dos dados de autenticação dos cidadãos da cidade do Rio de Janeiro.

Tecnologias​

  • OpenShift: Como solução de nuvem privada a Iplanrio efetuou a contratação da ferramenta OpenShift que é a distribuição kubernetes da RedHat. A partir deste ano, todos os projetos novos em Java são publicados e executados na plataforma Openshift.

Projetos Desenvolvidos​

  • Saude.rio APP
  • Backoffice Saude.rio
  • CADEVENTOS – Autorização para circulação de Eventos (Rock In Rio)
  • Identidade Carioca – id.rio

Fábrica de Software (Solutis)​

Foi contratada uma fábrica de software para ajudar a PCRJ no desenvolvimento de sistemas. A fábrica utilizou os padrões já utilizados no taxi.rio e saude.rio para desenvolver seus projetos. O Backend dos projetos foi desenvolvido em Java EE 7 e os frontends em ReactJS.

Os projetos da fábrica são executados no OpenShift.

Projetos da Fábrica de Software:**​

  • SINFRA – Gerencia os equipamentos da Secretaria de educação
  • SISBLOQTELE – Sistema para cadastro e bloqueio de ligação de telemarketing.
  • ACOLHERIO – Sistema para controle dos abrigos da PCRJ.

2020​

Nova Padronização​

No final de 2019 e início da 2020, a equipe java atuou em parceria com o time Mobile no desenvolvimento do aplicativo escola.rio. A equipe java ficou responsável pelo desenvolvimento do backend e por conta disso, iniciou-se a atualização da arquitetura para utilização do novo framework Quarkus.

A equipe java tambem atuou na publicação da plataforma Carioca Digital no Openshift e criou um componente em quarkus chamado gateway que efetua a i ntegração do catálogo de serviços Web em WordPress com o Identidade Carioca e a plataforma antiga que possui os serviços em Liferay.

Tecnologias​

  • Quarkus: A partir deste momento, todos os novos projetos de backend da equipe são criados utilizando Quarkus.

Maverick 4​

Após o fim do Jboss Seam, a equipe java não desenvolveu uma versão do Maverick para documentar os padrões e disponibilizar ferramentas de apoio. Com a consolidação de Quarkus e Docker como ferramentas essenciais no desenvolvimento de novos sistemas, estamos desenvolvendo a versão 4 da plataforma Maverick.

Essa versão disponibilizará novamente um site com a documentação de todos os padrões e uma ferramenta e site para criação de novos projetos.

Todos os projetos que não foram criados com Maverick 1 e 2 deverão ser migrados para o Maverick 4 para consolidar a padronização dos projetos.

Projetos Desenvolvidos​

  • Identidade Carioca
  • Carioca.rio
  • Escola.rio
  • Ficha Covid
  • Selo Covid
  • Lockdown
  • Cartão Alimentação SME
