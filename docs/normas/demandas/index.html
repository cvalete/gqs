Gestão de Demandas

O que é Gestão de Demanda de TI​

É um processo que tem o objetivo de identificar, analisar, priorizar,  e monitorar o atendimento às solicitações de soluções de tecnologia da informação para a IplanRio do início até o aceite do cliente.

Uma demanda que entra na IplanRio é uma pendência até que seja rejeitada ou tenha sido entregue. Ambas as situações devem ser acordadas com o cliente.

Definições​

O que é uma Demanda?​

É toda solicitação feita à Iplanrio que visa o desenvolvimento de um produto ou a prestação de um serviço​. ​ Para se cadastrar uma demanda ou aceitar uma demanda cadastrada pelo usuário no Jira temos que fazer uma pré-análise passando por pelo menos esses passos e se perguntando:

  1. Isso que chegou pra mim, é uma demanda comum?, ou seja uma demanda que já tem Projeto ou sistema associado e é ou já foi atendido pela Iplan?
  2. Mesmo já tendo projeto ou sistema associado, temos recursos disponíveis para atender?
  3. Ah não!!! Se você respondeu não a alguma das perguntas anteriores, precisa aprofundar a análise mais um pouquinho:
  • Necessita de avaliação  tempo de atendimento,  custo ou  alocação de equipe?
  • Necessita de estudo de viabilidade técnica, econômica e de recursos humanos?

O que é uma Atividade?​

Uma demanda pode ser resolvida com uma ou mais atividades

As atividades para resolução de uma demanda serão tratadas como subtarefas de uma pendência, ou seja, são todas as subtarefas que depois de concluídas, encerram a demanda.

A demanda estará pronta, quando todas as subtarefas forem concluídas. A utilização de subtarefas é interessante quando você tem partes do processo que dependem de pessoas diferentes e ou múltiplas etapas para conclusão de um processo.

Cadastramento de Jira​

Demandas sem projeto ou sistema pré-existente​

  1. Será criado um projeto com o nome Demandas.
  2. Para cada demanda abrir uma pendência no projeto Demandas de acordo com o fluxo do Processo de análise aceite da demanda;
  3. Todas as atividades referentes a pendência devem ser cadastradas no jira como sub-tarefas da  demanda inicial;
  4. As horas de trabalho alocadas devem ser documentadas  em cada subtarefa, serão alocadas na pendência apenas quando o atendimento a demanda corresponder a uma atividade, hipótese em que não será necessário cadastrar subtarefas.

Demandas com projeto ou sistema pré-existente​

  1. Para cada demanda abrir uma e só uma pendência no respectivo sistema/projeto de acordo com o fluxo do Processo de análise aceite da demanda;
  2. Se a para o atendimento da demanda for necessário realizar mais de uma atividade, deverão ser criadas subtarefas na pendência principal que corresponderá a demanda;
  3. As horas de trabalho alocadas devem ser documentadas  em cada subtarefa, serão alocadas na pendência apenas quando o atendimento a demanda corresponder a uma atividade, hipótese em que não será necessário cadastrar subtarefas.
